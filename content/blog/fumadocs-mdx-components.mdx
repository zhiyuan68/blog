# Introduction

This guide doubles as a live playground and a reference for the Fumadocs MDX component suite. It
covers the default Markdown syntax as well as the custom components that ship with
`fumadocs-ui`, so you can quickly copy and adapt the patterns for your own posts.

## Headings and paragraphs

Use standard Markdown headings (`#` through `######`) to create a clear document structure. The blog
layout automatically generates a table of contents from these headings, which appears to the left of
the article on desktop and above the content on mobile.

Paragraphs work as you would expect—just separate them with blank lines. You can mix **bold**,
_italic_, and `inline code` for emphasis.

## Lists and callouts

- Bullet lists are ideal for short tips.
- Ordered lists are perfect for linear tasks.
- Nested lists are also supported.

1. Start your project with `npm create fumadocs@latest`.
2. Pick the **blog** template.
3. Update the generated MDX files and push to your repository.

<Callout type="info" title="Reusable alerts">
  Callouts draw attention to important notes. Pass a `type` of `info`, `success`, `warning`, or
  `error` to switch the accent colour.
</Callout>

## Embedding images and media

Images use the standard Markdown syntax. You can add a `title` attribute if you want an optional
caption. When working offline, you can also drop down to raw HTML to specify the `width` and
`height` attributes explicitly:

<img
  src="https://images.unsplash.com/photo-1523475472560-d2df97ec485c?auto=format&fit=crop&w=1200&q=80"
  alt="A warm workspace illuminated by natural light"
  title="Workspace inspiration"
  loading="lazy"
  width="1200"
  height="800"
  className="rounded-2xl"
/>

Videos can be embedded with any React-friendly component. Here is a YouTube iframe, but you can
swap it for a `<video>` tag or a bespoke React player:

<iframe
  width="100%"
  height="360"
  src="https://www.youtube.com/embed/Ke90Tje7VS0"
  title="React in 100 Seconds"
  allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share"
  referrerPolicy="strict-origin-when-cross-origin"
  allowFullScreen
  className="rounded-xl border-none"
/>

## Code blocks and interactive tabs

Triple-backtick fences render syntax-highlighted code immediately:

```tsx
export function Hello() {
  return <span className="font-semibold">Hello, world!</span>;
}
```

When you need to compare multiple snippets, the code block tabs component keeps everything tidy:
